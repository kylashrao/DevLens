How the error location works

Rather than parsing the browser's built-in error message — which is worded differently in Chrome, Firefox, and Safari, and doesn't always include a position at all in newer versions — DevLens runs its own lightweight scanner over your text to find the exact line and column of the first structural problem. Validity itself is still confirmed with the browser's native JSON parser; the scanner is only used to locate where things went wrong.

Frequently asked questions

Why does my JSON key order change after formatting?

If your object has keys that look like integers (e.g. "1", "2"), JavaScript automatically reorders those keys numerically ahead of other keys. This is standard JavaScript object behavior, not a bug specific to this tool — it affects any formatter built on JSON.parse, which is nearly all of them. Regular string keys (like "name" or "email") always keep their original order.

Is there a size limit?

No hard limit — formatting and minifying use the browser's native JSON parser, which handles large documents well. Very large outputs may take slightly longer to syntax-highlight since each token is individually styled.

Does minify remove comments?

Standard JSON doesn't support comments, so if your input has any (as in JSONC or JSON5), it won't parse as valid JSON in the first place. Strip comments before pasting if your source uses one of those relaxed formats.
